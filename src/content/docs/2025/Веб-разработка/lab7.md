## Выполнение

### Чтение и отправка данных из файлов

Продолжите работу на ветке из прошлого занятия и допишите новый функционал.

В части backend укажите путь/пути до файлов html, css и js с вашей клиентской частью. Также добавьте еще одну страницу 404.html с заголовком об ошибке.

**Синхронно** считайте эти файлы в память с помощью модуля fs. Вы получите объект `Buffer`, что работать с ним как со строкой используйте метод `.toString()`. С помощью статического метода [Buffer.byteLength(string[, encoding])](https://nodejs.org/api/buffer.html#static-method-bufferbytelengthstring-encoding) посчитайте длину строки. Создайте объект `Map`, в котором ключами являются пути до файлов от корня, например, /index.html, а значениями вложенные объекты, в которых содержится само содержимое файла, его длина и тип контента (text/html, text/css).

Напишите функцию, которая из входящего запроса по пути ищет совпадение в объекте с содержимым файлов, при нахождении совпадений, подготавливает ответ устанавливая заголовки 'content-type', 'content-length' и тело ответа. Если путь указывает на корень /, то в ответе отправляйте index.html.

Протестируйте работу сервера с помощью браузера.

### Шаблонизация

В папке серверной части создайте папку templates. В ней разместите файл request-info.html. К этому файлу подключите style.css. В request-info.html добавьте разметку под информацию о пришедшем запросе. В шаблоне места, где должны появиться user-agent, accept-encoding и accept из заголовка запроса, выделите с двух сторон редко встречаемой комбинацией символов, например, %% имя %% или {{ имя }}.

Используйте модуль path для формирования путей к файлам. Например, path.join(\_\_dirname, 'public', 'index.html'). Файл request-info.html должен читаться **асинхронно** при каждом запросе, чтобы можно было без перезапуска сервера менять его структуру. Добавьте проверку на наличие файла перед его чтением с помощью fs.existsSync или fs.promises.access.

Данная страница должна возвращаться по GET-запросу на путь /request-info. Перед тем как ее отправить в ответе, замените с помощью метода строки replace обозначения на реальные данные из массива комментариев, выполнив минимальную санитизацию данных. Рассчитайте длину тела ответа, подготовьте заголовки и отправьте ответ. Если возникают проблемы с чтением файлов или заменой по шаблону, то сервер должен возвращать ошибку 500 Internal Server Error.

Протестируйте работу сервера с помощью браузера.


### Общая логика

1. Сервер должен последовательно искать совпадения путей. Сначала в объекте со статическими файлами, потом с шаблонами, затем по оставшимся вариантам. GET-запросы на любые другие пути должны передавать страницу 404 с соответствующим кодом статуса http.
1. Если вы сталкиваетесь с ситуацией, когда вы пишете схожую логику в разных местах проекта, подумайте как ее можно обобщить и выносите в отдельные вспомогательные функции и отдельные файлы.

### Нагрузочное тестирование

Добавьте в ваш проект файл с кодом ниже. Используйте его для проверки двух путей, через переменную URL. Протестируйте основные пути с get-запросами, выпишите результаты, оцените на сколько отличается обработка разных путей.

```js
import http from 'http';
import { performance } from 'node:perf_hooks';

const URL = 'http://localhost:3000/';
const CONCURRENCY = 50;   // parallel connections at once
const TOTAL = 5000;       // total requests to send

let completed = 0;
let errors = 0;
const start = performance.now();

function makeRequest() {
  return new Promise((resolve) => {
    const req = http.get(URL, (res) => {
      res.resume();
      res.on('end', resolve);
    });
    req.on('error', () => { errors++; resolve(); });
  });
}

async function runBatch() {
  const batch = [];
  for (let i = 0; i < CONCURRENCY; i++) {
    batch.push(makeRequest());
  }
  await Promise.all(batch);
  completed += CONCURRENCY;
}

(async () => {
  const batches = Math.ceil(TOTAL / CONCURRENCY);
  for (let i = 0; i < batches; i++) {
    await runBatch();
  }

  const elapsed = (performance.now() - start) / 1000; // seconds
  const rps = Math.round(completed / elapsed);

  console.log(`Completed : ${completed} requests`);
  console.log(`Errors    : ${errors}`);
  console.log(`Duration  : ${elapsed.toFixed(2)}s`);
  console.log(`Req/sec   : ${rps}`);
})();
```

## Требования к коду и результат выполнения

- Код отформатирован, сохранен в системе контроля версий.
- Отсутствуют ошибки от ESLint.
- Отсутствуют ошибки в консоли.
- Код разбит на логические модули.
