**Цель:** познакомиться с работой объектной модели документа, обработкой событий и элементом для отрисовки canvas.

## Изучить

### DOM

DOM как внутреннее представление браузером HTML-структуры документа в виде иерархии объектов, которыми можно манипулировать с помощью JavaScript.

Корневой, родительские, дочерние, одноуровневые узлы DOM, которые существуют относительно друг друга.

Navigator, Window и Document - важные части веб-браузера и то, как они представлены в JavaScript.

Получение ссылок на узлы DOM, например, с помощью querySelector() и getElementById().

Создание новых узлов, например, с помощью innerHTML() и createElement().

Добавление и удаление узлов в DOM с помощью appendChild() и removeChild().

Добавление атрибутов с помощью setAttribute().

Манипулирование стилями с помощью Element.style._ и Element.classList._.

### События

События — это сигналы, испускаемые браузером, когда происходит что-то важное, в ответ на которые разработчик может выполнить некоторый код.

Работа с обработчиками событий: addEventListener() и removeEventListener(), опции обработчика событий. Объекты событий.

Предотвращение поведения по умолчанию с помощью preventDefault().

Делегирование обработки событий благодаря всплытию.

### Canvas

Canvas (холст) для отображения растровой графики.

Получение контекста canvas и работа с ние.

Рисование примитивных объектов: fillRect, strokeRect, clearRect.

Работа с цветом: fillStyle и strokeStyle.

## Порядок выполнения лабораторной работы

В репозитории с прошлой работы переключитесь на ветку main. Добавьте файл README.md с ФИО и группой, сделайте коммит и отправьте ветку main на GitVerse.

Создайте ветку frontend и переключитесь на нее.

Создайте приложение типа Paint. Из его инструментов понадобятся:

- Карандаш
- Ластик (стерка)
- Рисование линий.
- Выбор цвета из заранее определенной палитры на 10 цветов (градации серого и оттенки красного, зеленого и синего).

Старайтесь реализовать 1 инструмент за раз. Пробуйте пользоваться paint или другим простым редактором для рисования, проанализируйте какие действия происходят в процессе использования инструмента и перенесите эту логику в вашу реализацию, совмещая обработку событий курсора, оценивание координат и вызов соответствующих функций для взаимодействия с контекстом canvas.

Протестируйте работу приложения и понятность для пользователя получившегося интерфейса. Можно для этого привлечь одногруппников.

Зафиксируйте результаты работы с помощью системы контроля версий git и отправьте ветку в репозиторий на GitVerse.
